chapter 1

프로그래밍 언어란?

사람이 작성한 코드로 해당 코드를 통해 기계의 동작을 제어하는것을 말한다

프로그래밍 언어에는 고급언어와 저급언어로 구분되는데, 고급언어는 사람이 쉽게 이해할수있는 언어로, 작성된 코드는 compile을 통해 기계어로 변환되어 사용된다. 저급언어는 기계어에 가까운 언어로, 대표적인 어셈블리어는 기계어와 1:1 대응되어 사용된다

프로그램(program)이란?

컴퓨터에서 특정 목적을 수행하기위해 프로그래밍 언어로 작성된 소스를 기계어로 번역한것을 말한다

자바란?

1995년 Sun Microsystems 에서 발표한 프로그래밍 언어로 엔터프라이즈 기업 환경에서 실행되는 서버 애플리케이션을 개발하는 중추적인 언어로 자리매김하고 있다.

자바의 특징

1. 이식성이 높은 언어이다

- JRE(Java Runtime Environment)가 설치되어 있는 모든 운영체제에서 실행가능하다

2. 객체지향 언어이다

- 캡슐화, 상속, 다형성 기능을 지원한다

3. 메모리 관리를 자동으로 해준다

- GC(Garbage Collector)을 실행시켜 사용하지않는 객체를 자동적으로 제거한다

4. 함수적 스타일 코딩을 지원한다

- 자바8부터 람다식(Lambda Expressions)을 지원, 컬렌션의 요소를 필터링, 매핑, 집계 처리하는데 쉬워졌다

5. Multti-Thread를 쉽게 구현할수 있다

- 여러작업 동시처리, 대용량작업을 위해 서브 작업으로 분리하여 병렬처리 할수 있는 Multti-Thread를 라이브러리 API 를 제공하여 쉽게 구현할수 있다

6. Dynamic Loading을 지원한다

- application이 실행될 때 모든객체가 생성되지않고, 객체가 필요한 시점에 클래스를 동적으로 로딩해서 객체를 생성한다

7. 오픈소스 라이브러리가 풍부하다

JVM(Java Virtual Machine)

JVM은 실운영체제를 대신해서 자바프로그램을 실행하는 가상의 운영체제 역활

OS와 자바프로그램을 중계하는 JVM을 두어 여러 운영체제에서 동일한 실행 결과를 만들수있다. 따라서 개발자는 운영체제와 상관없이 자바프로그램을 개발할수있다.

JRE(Java Runtime Environment) : JVM + 표준클래스라이브러리

JDK(Java Development Kit) : JRE + 개발에 필요한도구
